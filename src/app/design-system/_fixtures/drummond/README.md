# Sanitized fixtures (1.5b reference job)

This directory holds **sanitized** fixture data used by the `/design-system/prototypes/*` routes (Stage 1.5b).

The directory name (`drummond/`) labels the reference-job source per CLAUDE.md / MASTER-PLAN.md / VISION.md naming. Per CONTEXT D-21, the lowercase directory name is documentation labeling, not data, and is accepted leak. **Inside this directory, all data uses sanitized identifiers** (`caldwell-*` per CONTEXT D-27).

## Posture

- **All data here is COMMITTED** to git and shipped via Vercel preview. Hook T10c's sample-data isolation rule covers this directory by inheritance from `_fixtures/`.
- **Real customer data NEVER lives here.** Sanitization is mandatory per CONTEXT D-26 + D-27.
- The substitution map (real → fictional) lives at `.planning/fixtures/drummond/SUBSTITUTION-MAP.md` (gitignored).
- The raw source fixtures live at `.planning/fixtures/drummond/source3-downloads/` (gitignored).

## Three-tier grep gate

A three-tier grep gate rejects any committed `*.ts` file in this directory containing real customer identifiers from the locked substitution list:

1. **Tier 1 — extractor-side** (`scripts/sanitize-drummond.ts`) — runs grep against in-memory data before writing; refuses to write if real names survive substitution.
2. **Tier 2 — Claude pre-commit hook** (`.claude/hooks/nightwork-pre-commit.sh`) — blocks Claude-initiated `git commit` if any staged file in this path contains a high-risk identifier.
3. **Tier 3 — CI workflow** (`.github/workflows/drummond-grep-check.yml`) — final defense; blocks PRs and pushes to main.

All three tiers share the same pattern (locked substitution surnames + 17 vendor names + 9 Tier 2 customer surnames + 5 PM names; see CONTEXT D-26).

## Files (populated during /gsd-execute-phase)

- `index.ts` — barrel export
- `types.ts` — shared types (extends playground `_fixtures/` type pattern)
- `jobs.ts` — sanitized job record (1 entry; `j-caldwell-1`)
- `vendors.ts` — 17 sanitized vendors (`v-caldwell-*`)
- `cost-codes.ts` — sanitized cost code list (5-digit AIA G703 codes)
- `budget.ts` — sanitized budget lines (~25-50; `bl-caldwell-<cost-code>`)
- `change-orders.ts` — sanitized CO chain (4-6 entries; `co-caldwell-*`)
- `invoices.ts` — 4-6 representative invoices spanning the workflow (`inv-caldwell-*`)
- `draws.ts` — 5 historical pay apps with G703 detail (`d-caldwell-01` through `d-caldwell-05`)
- `draw-line-items.ts` — one per cost code per draw (~100+ rows total)
- `lien-releases.ts` — Florida 4-statute lien release set (`lr-caldwell-*`)
- `schedule.ts` — reconstructed schedule for Gantt prototype (per Q2 override C; `s-caldwell-*`)
- `payments.ts` — derived payment records (`p-caldwell-*`)
- `reconciliation.ts` — 8 paired imported/current shapes (4 candidates × 2 drift types; `rec-caldwell-*`)


This README lives here permanently — it documents the directory's purpose for future readers without leaking real customer identifiers (per the three-tier grep gate it would otherwise trip).
